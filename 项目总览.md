## 🎯 系统功能

### 核心能力

✅ **全自动化**：一键运行，自动完成数据加载→特征提取→评估→诊断→报告生成  
✅ **智能诊断**：自动定位3类原因（识别错误、打分公式、融合策略）  
✅ **多格式输出**：JSON（程序处理）+ HTML（可视化）+ CSV（Excel分析）  
✅ **高度可配置**：30+ 阈值参数可通过配置文件调整  
✅ **详细日志**：完整记录分析过程，便于调试

## 🏗️ 系统架构

### 五大核心模块

```
1️⃣ DataPreprocessor        数据预处理
   ├─ 加载JSON/CSV数据
   ├─ 计算doc年龄（天/小时）
   └─ 数据格式验证

2️⃣ FeatureExtractor        特征提取
   ├─ 计算30+ 特征
   ├─ TOP-K统计（K=3,5,10）
   ├─ 时间窗口分析（1,3,7,30天）
   └─ Baseline特征（engine1&2平均）

3️⃣ TimelinessEvaluator     时效性评估
   ├─ 4维度评分
   │  ├─ 平均doc年龄（40%）
   │  ├─ 新鲜文档覆盖率（30%）
   │  ├─ 最新文档发现（20%）
   │  └─ 时效卡召回（10%）
   └─ 综合得分 0-100

4️⃣ ReasonDiagnoser         原因诊断
   ├─ 原因1：识别错误（P0）
   ├─ 原因2：打分公式（P1）
   └─ 原因3：融合策略（P2）

5️⃣ ReportGenerator         报告生成
   ├─ JSON报告（完整数据）
   ├─ HTML报告（可视化）
   └─ CSV报告（Excel友好）
```

## 📊 评估指标体系

### 时效性评分公式

```
总分 = 维度1 × 40% + 维度2 × 30% + 维度3 × 20% + 维度4 × 10%

维度1：平均doc年龄得分
  - test与baseline的TOP5平均doc年龄比值
  - 比值 ≤1.0 → 100分
  - 比值 >2.0 → 接近0分

维度2：新鲜文档覆盖率得分
  - 7天内文档占比差值
  - 差值 ≥0 → 100分
  - 差值 <-0.4 → 接近0分

维度3：最新文档发现能力得分
  - 最小doc年龄比值
  - 比值 ≤1.2 → 100分
  - 比值 >5.0 → 0分

维度4：时效卡得分
  - 简化处理（基于其他维度）
```

### 判定标准

| 得分范围 | 标签 | 说明 |
|---------|------|-----|
| ≥70 | ✅ 时效性良好 | 无需处理 |
| 50-70 | ⚠️ 时效性一般 | 可观察 |
| <50 | ❌ 时效性差 | **需要诊断原因** |

---

## 🔍 诊断逻辑

### 三层原因诊断（按优先级）

#### 原因1：Query时效性等级识别错误 [P0]

**触发条件**（满足任一）：
1. 两个权威引擎都有时效卡 → 应该是"高时效"
2. 一个引擎有卡 + 平均doc年龄≤7天 → 应该是"高时效"
3. 平均doc年龄≤3天 + 新鲜文档占比≥80% → 应该是"高时效"

**但test判定为"其他时效"** → 识别错误

---

#### 原因2：Doc时效性打分公式不合理 [P1]

**检查项**（任一异常即触发）：

| 检查项 | 高时效阈值 | 其他时效阈值 |
|-------|-----------|-------------|
| ① 相关系数 | -1.0 ~ -0.6 | -0.6 ~ -0.2 |
| ② 打分标准差 | ≥0.15 | ≥0.05 |
| ③ 新旧文档打分差 | ≥0.3 | ≥0.1 |
| ④ 新文档排序位置 | 不低于baseline的1.5倍 | |

---

#### 原因3：融合策略不合理 [P2]

**检查项**（任一异常即触发）：
1. 理想排序提升潜力 >30%
2. 新文档时效分高，但<50%进入TOP10
3. 高时效query的新鲜度提升 < baseline的50%

## 🚀 使用流程

### 基础用法（3步）

```bash
# 1. 安装依赖
pip install -r requirements.txt

# 2. 运行分析
python3 main.py --input your_data.json --output ./results

# 3. 查看报告
open results/analysis_report.html
```

### 高级用法

```bash
# 自定义配置
python3 main.py --input data.json --config my_config.yaml --output ./results

# 批量处理
for file in data/*.json; do
    python3 main.py --input "$file" --output "results/$(basename $file .json)"
done
```

## ⚙️ 配置参数说明

### 核心阈值

```yaml
thresholds:
  poor_score: 50      # 时效性差阈值（建议：40-60）
  fair_score: 70      # 时效性一般阈值（建议：60-80）
  
  weights:            # 评分权重（总和=1.0）
    avg_age: 0.4      # 平均doc年龄权重
    fresh_ratio: 0.3  # 新鲜文档占比权重
    min_age: 0.2      # 最新文档发现权重
    card: 0.1         # 时效卡权重
```

### 诊断阈值

```yaml
  recognition:              # 原因1阈值
    fresh_age_strong: 3     # 强新鲜度（天）
    fresh_age_medium: 7     # 中等新鲜度（天）
    fresh_ratio_high: 0.8   # 高集中度
    
  scoring:                  # 原因2阈值
    high_timeliness:
      corr_min: -1.0        # 相关系数范围
      corr_max: -0.6
      std_min: 0.15         # 最小标准差
      gap_min: 0.3          # 新旧文档差距
      
  fusion:                   # 原因3阈值
    improvement_potential: 0.3  # 提升潜力
    fresh_suppression_ratio: 0.5  # 压制比例
```

## 💡 最佳实践

### 1. 数据准备

- ✅ 确保时间格式统一：`2025-10-15T10:00:00`
