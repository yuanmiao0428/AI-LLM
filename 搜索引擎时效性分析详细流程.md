## 二、时效性表现评估体系

### 2.1 建立权威基准

基于引擎1和引擎2，构建"权威时效性基准"：

```python
# 对每个query，计算权威引擎的平均表现
baseline_avg_doc_age_top5 = (engine1_avg_doc_age_top5 + engine2_avg_doc_age_top5) / 2
baseline_fresh_ratio_7d = (engine1_fresh_ratio_7d + engine2_fresh_ratio_7d) / 2
baseline_min_doc_age = min(engine1_min_doc_age, engine2_min_doc_age)

# 时效卡标签：只要引擎1或引擎2有时效卡，就认为该query有时效性需求
baseline_has_timeliness_need = engine1_has_timeliness_card OR engine2_has_timeliness_card
```

### 2.2 时效性表现差的判定标准

#### 评分机制：为每个query计算时效性得分（0-100分）

##### 2.2.1 维度1：平均doc年龄得分（权重40%）

```python
# 计算引擎test相对于baseline的表现
age_ratio = test_avg_doc_age_top5 / baseline_avg_doc_age_top5

# 评分规则
if age_ratio <= 1.0:
    age_score = 100  # test表现好于或等于baseline
elif age_ratio <= 1.5:
    age_score = 100 - (age_ratio - 1.0) * 100  # 线性递减
elif age_ratio <= 2.0:
    age_score = 50 - (age_ratio - 1.5) * 60
else:
    age_score = max(0, 20 - (age_ratio - 2.0) * 20)

维度1得分 = age_score * 0.4
```

**阈值说明**：
- `age_ratio <= 1.0`：test不差于baseline，满分
- `1.0 < age_ratio <= 1.5`：轻微劣势，50-100分
- `1.5 < age_ratio <= 2.0`：明显劣势，20-50分
- `age_ratio > 2.0`：严重劣势，0-20分

##### 2.2.2 维度2：新鲜文档覆盖率得分（权重30%）

```python
# 使用7天窗口
fresh_diff = test_fresh_ratio_7d - baseline_fresh_ratio_7d

# 评分规则
if fresh_diff >= 0:
    fresh_score = 100  # test表现好于或等于baseline
elif fresh_diff >= -0.2:
    fresh_score = 100 + fresh_diff * 250  # 从100分线性递减
elif fresh_diff >= -0.4:
    fresh_score = 50 + (fresh_diff + 0.2) * 150
else:
    fresh_score = max(0, 50 + (fresh_diff + 0.4) * 100)

维度2得分 = fresh_score * 0.3
```

**阈值说明**：
- `fresh_diff >= 0`：test不差于baseline，满分
- `-0.2 <= fresh_diff < 0`：轻微劣势（如baseline 50%，test 30%），50-100分
- `-0.4 <= fresh_diff < -0.2`：明显劣势，20-50分
- `fresh_diff < -0.4`：严重劣势，0-20分

##### 2.2.3 维度3：最新文档发现能力得分（权重20%）

```python
# 检查test能否找到足够新的文档
min_age_ratio = test_min_doc_age / baseline_min_doc_age

# 评分规则
if min_age_ratio <= 1.2:
    min_age_score = 100  # test能找到同样新的文档
elif min_age_ratio <= 2.0:
    min_age_score = 100 - (min_age_ratio - 1.2) * 62.5
elif min_age_ratio <= 5.0:
    min_age_score = 50 - (min_age_ratio - 2.0) * 13.3
else:
    min_age_score = 0

维度3得分 = min_age_score * 0.2
```

**阈值说明**：
- `min_age_ratio <= 1.2`：test能找到相近新鲜度的文档，满分
- `1.2 < min_age_ratio <= 2.0`：轻微劣势，50-100分
- `2.0 < min_age_ratio <= 5.0`：明显劣势，10-50分
- `min_age_ratio > 5.0`：完全找不到新文档，0分

##### 2.2.4 维度4：时效卡召回得分（权重10%）

```python
# 只有当baseline认为有时效性需求时才计算
if baseline_has_timeliness_need:
    # 注意：test没有时效卡数据，通过其他维度间接评估
    # 这里简化为：如果前三个维度得分都低，说明可能缺少时效卡
    card_score = 0 if (维度1得分 + 维度2得分 + 维度3得分) < 40 else 100
else:
    card_score = 100  # 不需要时效卡，不扣分

维度4得分 = card_score * 0.1
```

##### 2.2.5 综合时效性得分

```python
timeliness_score = 维度1得分 + 维度2得分 + 维度3得分 + 维度4得分

# 判定标准
if timeliness_score >= 70:
    label = "时效性良好"
elif timeliness_score >= 50:
    label = "时效性一般"
else:
    label = "时效性差"
```

**核心阈值**：
- **时效性差**：`timeliness_score < 50`（需要重点分析）
- **时效性一般**：`50 <= timeliness_score < 70`（可观察）
- **时效性良好**：`timeliness_score >= 70`

## 三、原因诊断流程

对于所有`timeliness_score < 50`的query，进行三层原因诊断。

### 3.1 【原因1】Query时效性等级识别错误

#### 3.1.1 诊断指标

**指标1.1：时效性需求置信度**

基于引擎1和引擎2的时效卡情况：

```python
# 置信度评估
if engine1_has_timeliness_card and engine2_has_timeliness_card:
    timeliness_need_confidence = "高"  # 两个权威引擎都有时效卡
elif engine1_has_timeliness_card or engine2_has_timeliness_card:
    timeliness_need_confidence = "中"  # 一个权威引擎有时效卡
else:
    timeliness_need_confidence = "低"  # 都没有时效卡
```

**指标1.2：权威引擎新鲜度倾向**

```python
# 如果权威引擎平均返回非常新的文档，说明query有时效性
baseline_avg_doc_age_top5 = (engine1_avg + engine2_avg) / 2

if baseline_avg_doc_age_top5 <= 3:  # 3天以内
    freshness_preference = "强"
elif baseline_avg_doc_age_top5 <= 7:  # 7天以内
    freshness_preference = "中"
elif baseline_avg_doc_age_top5 <= 30:  # 30天以内
    freshness_preference = "弱"
else:
    freshness_preference = "无"
```

**指标1.3：权威引擎新鲜文档占比**

```python
if baseline_fresh_ratio_7d >= 0.8:  # TOP5中80%是7天内的文档
    fresh_concentration = "高"
elif baseline_fresh_ratio_7d >= 0.5:
    fresh_concentration = "中"
else:
    fresh_concentration = "低"
```

#### 3.1.2 判定逻辑

```python
# 判定query应该是"高时效"
should_be_high_timeliness = False

if timeliness_need_confidence == "高":
    should_be_high_timeliness = True
elif timeliness_need_confidence == "中" and freshness_preference in ["强", "中"]:
    should_be_high_timeliness = True
elif freshness_preference == "强" and fresh_concentration == "高":
    should_be_high_timeliness = True

# 识别错误判定
if should_be_high_timeliness and test_query_timeliness_level == "其他时效":
    reason1_flag = True
    reason1_detail = {
        "错误类型": "高时效query被误判为其他时效",
        "置信度": timeliness_need_confidence,
        "权威引擎平均doc年龄": baseline_avg_doc_age_top5,
        "权威引擎新鲜文档占比": baseline_fresh_ratio_7d,
        "引擎1时效卡": engine1_has_timeliness_card,
        "引擎2时效卡": engine2_has_timeliness_card,
        "test判别": test_query_timeliness_level
    }
else:
    reason1_flag = False
```

#### 3.1.3 阈值总结

| 指标 | 阈值 | 说明 |
|-----|------|-----|
| 时效卡置信度 | 两个引擎都有卡 | 高置信度认为是高时效query |
| 平均doc年龄 | ≤3天 | 强新鲜度偏好 |
| 平均doc年龄 | 3-7天 | 中等新鲜度偏好 |
| 新鲜文档占比 | ≥80% | 高集中度 |
| 新鲜文档占比 | 50%-80% | 中等集中度 |

### 3.2 【原因2】Doc时效性打分公式不合理

**前提条件**：Query时效性等级识别正确（reason1_flag = False）

#### 3.2.1 诊断指标

**指标2.1：Doc年龄与时效性打分的相关性**

```python
# 计算Pearson相关系数
correlation = pearson_corr(doc_age_list, timeliness_score_list)

# 评估标准（应该是负相关）
if test_query_timeliness_level == "高时效":
    expected_corr_range = (-1.0, -0.6)  # 强负相关
else:
    expected_corr_range = (-0.6, -0.2)  # 弱到中等负相关

# 判定
if correlation < expected_corr_range[0] or correlation > expected_corr_range[1]:
    corr_abnormal = True
```

**阈值说明**：
- 高时效query：相关系数应在-0.6到-1.0之间（强负相关）
- 其他时效query：相关系数应在-0.2到-0.6之间（弱到中等负相关）

**指标2.2：时效性打分区分度**

```python
# 计算打分标准差
score_std = std(timeliness_score_list)

# 评估标准
if test_query_timeliness_level == "高时效":
    min_expected_std = 0.15  # 高时效应该有较大区分度
else:
    min_expected_std = 0.05  # 其他时效区分度较小

# 判定
if score_std < min_expected_std:
    low_discrimination = True
```

**阈值说明**：
- 高时效query：打分标准差应≥0.15（假设打分范围0-1）
- 其他时效query：打分标准差应≥0.05

**指标2.3：新旧文档打分对比**

```python
# 将文档分为新、旧两组
fresh_docs = [doc for doc in docs if doc.age <= 7]
old_docs = [doc for doc in docs if doc.age > 30]

if len(fresh_docs) > 0 and len(old_docs) > 0:
    avg_fresh_score = mean([doc.timeliness_score for doc in fresh_docs])
    avg_old_score = mean([doc.timeliness_score for doc in old_docs])
    
    score_gap = avg_fresh_score - avg_old_score
    
    # 评估标准
    if test_query_timeliness_level == "高时效":
        min_expected_gap = 0.3  # 新旧文档打分差距应该大
    else:
        min_expected_gap = 0.1
    
    # 判定
    if score_gap < min_expected_gap:
        insufficient_gap = True
```

**阈值说明**：
- 高时效query：新文档（≤7天）与旧文档（>30天）打分差≥0.3
- 其他时效query：打分差≥0.1

**指标2.4：打分衰减速度分析**

```python
# 按doc年龄分桶，计算每桶的平均打分
buckets = {
    "0-1天": [],
    "1-3天": [],
    "3-7天": [],

    "30天以上": []
}

# 计算衰减率

# 评估标准（高时效query）
if test_query_timeliness_level == "高时效":
    # 前期应该快速衰减
    if decay_rate_0_3 < 0.1:  # 0-3天衰减不足10%

    # 后期衰减应该放缓
    if decay_rate_3_7 > decay_rate_0_3:  # 后期比前期衰减还快
        abnormal_decay_pattern = True
```

**阈值说明**：
- 高时效query：0-3天内打分应衰减≥10%
- 衰减速度应该递减（前期快，后期慢）

**指标2.5：对比权威引擎的doc年龄-排序关系**

```python
# 虽然doc不同，但可以对比统计规律
# 计算权威引擎中，新文档的平均排序位置
engine1_fresh_avg_rank = mean([rank for doc, rank in engine1_docs if doc.age <= 7])
```

**阈值说明**：
- test中新文档的平均排名 不应超过 baseline的1.5倍

### 3.3 【原因3】Doc时效性打分融合策略不合理

**前提条件**：
- Query时效性等级识别正确（reason1_flag = False）
- Doc时效性打分公式合理（reason2_flag = False）
- 但整体时效性表现仍然差（timeliness_score < 50）

#### 3.3.1 诊断指标

**指标3.1：时效性打分与最终排序的相关性**

由于没有最终排序数据，用以下方法推断：

```python
# 假设：如果时效性打分合理，但整体表现差，说明融合时权重不够

# 构造"理想排序"：纯按时效性打分排序
ideal_rank_by_timeliness = rank_docs_by_score(test_docs, 'timeliness_score')

# 计算理想排序下的时效性表现
ideal_avg_doc_age_top5 = avg([doc.age for doc in ideal_rank_by_timeliness[:5]])
ideal_fresh_ratio_7d = count([doc.age <= 7 for doc in ideal_rank_by_timeliness[:5]]) / 5

# 对比当前实际表现（当前排序未知，假设按返回顺序）
actual_avg_doc_age_top5 = test_avg_doc_age_top5
actual_fresh_ratio_7d = test_fresh_ratio_7d

# 判定
if ideal_avg_doc_age_top5 < actual_avg_doc_age_top5 * 0.7:
    # 如果理想排序能大幅改善，说明融合策略有问题
    fusion_weight_too_low = True
    improvement_potential = (actual_avg_doc_age_top5 - ideal_avg_doc_age_top5) / actual_avg_doc_age_top5
```

**阈值说明**：
- 如果纯按时效性打分排序，能使平均doc年龄降低30%以上，说明融合权重过低

**指标3.2：高时效query的融合策略区分**

```python
# 对比高时效query和其他时效query的表现差异

# 计算所有高时效query的平均表现提升空间
high_timeliness_queries_improvement = []
other_timeliness_queries_improvement = []

for query in all_queries:
    improvement = (actual_avg_age - ideal_avg_age) / actual_avg_age
    if query.test_level == "高时效":
        high_timeliness_queries_improvement.append(improvement)
    else:
        other_timeliness_queries_improvement.append(improvement)

avg_high_improvement = mean(high_timeliness_queries_improvement)
avg_other_improvement = mean(other_timeliness_queries_improvement)

# 判定
if avg_high_improvement > 0.3 and avg_high_improvement > avg_other_improvement * 1.5:
    # 高时效query的融合权重明显偏低
    high_timeliness_fusion_issue = True
```

**阈值说明**：
- 高时效query的提升空间>30%，且明显高于其他时效query，说明融合策略未区分query类型

**指标3.3：新文档召回但未排序靠前**

```python
# 检查新文档的召回和排序情况
fresh_docs_in_top10 = [doc for doc in test_docs[:10] if doc.age <= 7]
fresh_docs_in_all = [doc for doc in test_docs if doc.age <= 7]

# 计算新文档的时效性打分
if len(fresh_docs_in_all) >= 3:

    all_docs_avg_score = mean([doc.timeliness_score for doc in test_docs])
    
    # 如果新文档的时效性打分高，但在top10中占比少
    if fresh_docs_avg_score > all_docs_avg_score + 0.1:
        # 新文档打分确实高
        if len(fresh_docs_in_top10) < len(fresh_docs_in_all) * 0.5:
```

```python
            'fair_timeliness_score': 70,
            
            # 原因1：识别错误阈值
            'fresh_age_strong': 3,  # 天
            'fresh_age_medium': 7,
            'fresh_ratio_high': 0.8,
            'fresh_ratio_medium': 0.5,
            
            # 原因2：打分公式阈值
            'high_timeliness_corr_range': (-1.0, -0.6),
            'other_timeliness_corr_range': (-0.6, -0.2),
            'high_timeliness_std': 0.15,
            'other_timeliness_std': 0.05,
            'high_timeliness_gap': 0.3,
            'other_timeliness_gap': 0.1,
            'decay_rate_min': 0.1,
            'fresh_rank_ratio_max': 1.5,
            
            # 原因3：融合策略阈值
            'improvement_potential_threshold': 0.3,
            'high_query_boost_threshold': 0.2,
            'fresh_suppression_ratio': 0.5,
        }
    
    def analyze_query(self, query_data):
        """分析单个query的时效性表现"""
        result = {
            'query': query_data['query'],
            'timeliness_score': 0,
            'label': '',
            'reasons': []
        }
        
        # Step 1: 计算特征
        features = self.compute_features(query_data)
        
        # Step 2: 评估时效性表现
        score = self.evaluate_timeliness(features)
        result['timeliness_score'] = score
        
        if score >= self.THRESHOLDS['fair_timeliness_score']:
            result['label'] = '时效性良好'
            return result
        elif score >= self.THRESHOLDS['poor_timeliness_score']:
            result['label'] = '时效性一般'
        else:
            result['label'] = '时效性差'
        
        # Step 3: 诊断原因（只针对时效性差的case）
        if score < self.THRESHOLDS['poor_timeliness_score']:
            # 原因1：识别错误
            reason1 = self.diagnose_recognition_error(query_data, features)
            if reason1['flag']:
                result['reasons'].append(reason1)
                return result  # 识别错误是根本原因，直接返回
            
            # 原因2：打分公式
            reason2 = self.diagnose_scoring_formula(query_data, features)
            if reason2['flag']:
                result['reasons'].append(reason2)
                return result  # 打分公式问题，直接返回
            
            # 原因3：融合策略
            reason3 = self.diagnose_fusion_strategy(query_data, features)
            if reason3['flag']:
                result['reasons'].append(reason3)
        
        return result
    
    def compute_features(self, query_data):
        """计算各项特征"""
        features = {}
        
        # 计算引擎test特征
        test_docs = query_data['test_docs']
        features['test_avg_doc_age_top5'] = self._avg_doc_age(test_docs[:5])
        features['test_fresh_ratio_7d'] = self._fresh_ratio(test_docs[:5], 7)
        features['test_min_doc_age'] = min([d['age'] for d in test_docs])
        # ... 更多特征
        
        # 计算baseline特征
        e1_docs = query_data['engine1_docs']
        e2_docs = query_data['engine2_docs']
        features['baseline_avg_doc_age_top5'] = (
            self._avg_doc_age(e1_docs[:5]) + self._avg_doc_age(e2_docs[:5])
        ) / 2

    def evaluate_timeliness(self, features):

        # 维度1：平均doc年龄得分
        age_ratio = features['test_avg_doc_age_top5'] / features['baseline_avg_doc_age_top5']
        score1 = self._calc_age_score(age_ratio) * 0.4

        score2 = self._calc_fresh_score(fresh_diff) * 0.3
        
        # 维度3：最新文档发现能力得分
        min_age_ratio = features['test_min_doc_age'] / features['baseline_min_doc_age']
        score3 = self._calc_min_age_score(min_age_ratio) * 0.2
        
        # 维度4：时效卡得分
        score4 = self._calc_card_score(features) * 0.1
        
        return score1 + score2 + score3 + score4
```
