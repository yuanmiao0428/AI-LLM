🚀 快速开始（3步）
--------------------------------------------------------------------------------

1️⃣ 安装依赖
   pip install pandas numpy pyyaml

2️⃣ 运行测试（验证系统）
   python3 test_system.py
   
3️⃣ 运行分析
   python3 main.py --input sample_data.json --output ./results
   
   然后查看结果：
   - results/analysis_report.html  (浏览器打开)
   - results/poor_queries.csv      (Excel打开)
   - results/analysis_report.json  (程序处理)

📊 系统功能
--------------------------------------------------------------------------------
✅ 自动评估每个query的时效性得分（0-100分）
✅ 自动诊断3类原因：
   - 原因1：Query时效性等级识别错误 [P0]
   - 原因2：Doc时效性打分公式不合理 [P1]
   - 原因3：融合策略不合理 [P2]
✅ 生成多格式报告（JSON + HTML + CSV）

🎯 使用场景
--------------------------------------------------------------------------------

场景1: 快速诊断当前数据
   python3 main.py --input your_data.json --output ./results

场景2: 使用自定义配置
   python3 main.py --input data.json --config my_config.yaml --output ./results

场景3: 批量处理多个文件
   for file in data/*.json; do
       python3 main.py --input "$file" --output "results/$(basename $file .json)"
   done

📋 数据格式要求
--------------------------------------------------------------------------------
JSON格式，每个query包含：
{
  "query": "查询文本",
  "search_time": "2025-10-15T10:00:00",
  "engine1": {
    "docs": [{"url": "...", "title": "...", "publish_time": "..."}],
    "has_timeliness_card": true/false
  },
  "engine2": { 同engine1 },
  "test": {
    "docs": [{"url": "...", "title": "...", "publish_time": "...", "timeliness_score": 0.85}],
    "query_timeliness_level": "高时效" / "其他时效"
  }
}

详细格式参考: sample_data.json

⚙️ 配置说明
--------------------------------------------------------------------------------
所有阈值在 config.yaml 中配置：

- poor_score: 50        时效性差的阈值（<50分需要诊断）
- fair_score: 70        时效性一般的阈值（50-70分）

评分权重（总和=1.0）：
- avg_age: 0.4          平均doc年龄权重
- fresh_ratio: 0.3      新鲜文档占比权重
- min_age: 0.2          最新文档发现权重
- card: 0.1             时效卡权重

根据实际需求调整阈值，重新运行分析即可。

🔍 结果解读
--------------------------------------------------------------------------------

时效性评分：
  ≥70分  → ✅ 时效性良好（无需处理）
  50-70分 → ⚠️ 时效性一般（可观察）
  <50分  → ❌ 时效性差（需要分析原因）

原因类型：
  原因1_识别错误   → query时效性等级判断错误（最高优先级P0）
  原因2_打分公式   → doc时效性打分计算不合理（优先级P1）
  原因3_融合策略   → 时效性打分融合权重过低（优先级P2）

💡 改进建议
--------------------------------------------------------------------------------

发现问题后，按优先级处理：

P0 - 识别错误：
  - 检查被误判的query特征
  - 优化时效性等级识别模型
  - 增加时效性关键词特征

P1 - 打分公式：
  - 分析doc年龄与打分的关系
  - 调整衰减函数参数
  - 增大新旧文档打分差距

P2 - 融合策略：
  - 提升时效性打分权重
  - 针对高时效query使用更高权重
  - A/B测试验证效果

❓ 常见问题
--------------------------------------------------------------------------------

Q: 运行出错提示缺少模块？
A: 运行: pip install pandas numpy pyyaml

Q: 所有query都是"时效性良好"？
A: 阈值可能过低，调整config.yaml中的poor_score和fair_score

Q: 想看更详细的日志？
A: 修改config.yaml中的logging.level为DEBUG

Q: 数据格式有问题？
A: 参考sample_data.json，确保时间格式为ISO 8601

Q: 如何提取特定原因的query？
A: 导入JSON报告，按diagnosis.reason字段筛选
