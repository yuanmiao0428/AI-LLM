## 三、运行分析（3步骤）

### 第1步：准备数据

将您的数据保存为JSON格式（参考 `sample_data.json`）：

```json

  {

    "search_time": "2025-10-15T10:00:00",

    "engine2": {...},
    "test": {

      "query_timeliness_level": "高时效"
```


## 四、结果解读

### 时效性评分

- **≥70分**: ✅ 时效性良好（无需处理）
- **50-70分**: ⚠️ 时效性一般（可观察）
- **<50分**: ❌ 时效性差（需要分析原因）

### 原因分类

| 原因 | 优先级 | 说明 |
|-----|--------|-----|
| 原因1_识别错误 | P0 | query时效性等级判断错误 |
| 原因2_打分公式 | P1 | doc时效性打分计算不合理 |
| 原因3_融合策略 | P2 | 时效性打分融合权重过低 |

## 五、常见使用场景

### 场景1：快速诊断

```bash
# 使用默认配置快速分析
python3 main.py --input data.json --output ./results
```

### 场景2：自定义阈值

编辑 `config.yaml`：

```yaml
thresholds:
  poor_score: 60  # 更严格的标准
  weights:
    avg_age: 0.5  # 增加年龄权重
```

然后运行：

```bash
python3 main.py --input data.json --output ./results --config config.yaml
```

### 场景3：只关注高时效query

分析后，从CSV文件中筛选：

```python
import pandas as pd

df = pd.read_csv('results/poor_queries.csv')
high_timeliness = df[df['test_query_level'] == '高时效']
print(high_timeliness)
```

## 七、下一步行动

根据分析结果，按优先级改进：

### P0：修复识别错误

如果大量query被识别错误：

1. 检查这些query的共同特征
2. 优化时效性等级识别模型
3. 增加时效性关键词（如"最新"、"今天"等）

### P1：调整打分公式

如果打分公式不合理：

1. 分析doc年龄与打分的关系曲线
2. 调整衰减函数参数
3. 增大新旧文档打分差距

### P2：优化融合策略

如果融合权重过低：

1. 提升时效性打分权重
2. 针对高时效query使用不同权重
3. A/B测试验证效果

## 九、FAQ

**Q: 所有query都显示"时效性良好"？**

A: 阈值可能过低，调整 `config.yaml` 中的 `poor_score` 和 `fair_score`

**Q: 想看更详细的日志？**

A: 修改 `config.yaml` 中的 `logging.level: DEBUG`

**Q: 数据格式错误怎么办？**

A: 参考 `sample_data.json`，确保时间格式为 `2025-10-15T10:00:00`

**Q: 可以分析CSV格式的数据吗？**

A: 可以，但建议使用JSON格式，更易处理嵌套结构
